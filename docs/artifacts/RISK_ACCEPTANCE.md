# RISK_ACCEPTANCE

## Accepted in current product direction
1. **Inference risk** from overlapping filter comparisons is accepted.
2. **Comment self-identification risk** is accepted when comments are enabled.
3. **Local-only daily lock bypass risk** is accepted for stronger privacy posture.

## User disclosure obligations
- Explain accepted risks in product documentation and verification UI.
- Do not present accepted-risk areas as hard anonymity guarantees.

## Revisit triggers
- Enterprise customer requirement change
- Incident involving re-identification
- Regulatory/legal requirement update
